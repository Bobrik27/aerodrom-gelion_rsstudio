# Аудит сайта-предшественника (aerodrom-gelion.ru)

**Дата анализа:** 2025-11-03

Этот документ фиксирует состояние исходного сайта для понимания его недостатков и определения целей редизайна.

---

## 1. Ключевые проблемы

### 1.1. Техническое состояние
- **Критическая производительность:** Время ответа сервера (TTFB) составляет ~34 секунды. Это делает сайт непригодным для использования.
- **Отсутствие оптимизации:** Ресурсы (CSS, JS) блокируют отрисовку. Изображения не оптимизированы.
- **Устаревший стек:** Статические HTML-файлы без системы управления или современного фреймворка.
- **Неадаптивность:** Сайт не оптимизирован для мобильных устройств.

### 1.2. Структура и навигация
- **Плоская структура:** Набор несвязанных `.html` файлов.
- **Негибкость:** Добавление нового контента требует ручного редактирования множества файлов.
- **Недружелюбные URL:** Наличие `.html` в адресах страниц.

### 1.3. Контент и UX
- **Пассивность:** Сайт является пассивной "визиткой", а не инструментом для привлечения клиентов.
- **Отсутствие призывов к действию (CTA):** Нет форм для заявок, кнопок заказа и т.д.
- **Низкая интерактивность:** Отсутствуют любые динамические элементы (карты, калькуляторы).

---

## 2. Цели редизайна (Что мы строим)

1.  **Скорость:** Добиться загрузки страниц менее чем за 2 секунды (TTFB < 500ms).
2.  **Современный стек:** Использовать Next.js, React и TypeScript для масштабируемости и удобства разработки.
3.  **Mobile-First дизайн:** Обеспечить идеальное отображение и функциональность на всех устройствах с помощью Tailwind CSS.
4.  **SEO-оптимизация:** Создать семантически верную структуру с чистыми URL для лучшего ранжирования в поисковых системах.
5.  **Ориентация на конверсию:** Внедрить понятные призывы к действию, формы обратной связи и интерактивные элементы для превращения посетителей в клиентов.
6.  **Масштабируемость:** Заложить архитектуру, которая позволит легко добавлять новые разделы и функционал (блог, личный кабинет, системы бронирования).
